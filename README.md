# Obsidian PDF Chat

Chat with selected text and PDFs in Obsidian using your own OpenAI-compatible API.

This plugin is designed for academic paper reading: select text in a PDF, press a hotkey, type a question, and get the answer in a movable/resizable chat modal.

## Features

- Ask questions about selected text from PDF or other selectable views.
- Press `Enter` to submit; use `Shift+Enter` for a new line.
- Streaming responses with a stop button.
- Multi-turn conversation memory.
- Model selector in the modal and configurable model list in settings.
- Reading presets for paper reading workflows.
- Optional full-paper summary as background context.
- For medium-length PDFs, reads the full extracted text once per conversation for better accuracy.
- For long PDFs, falls back to local BM25 keyword retrieval with query expansion.
- Markdown and formula rendering support.
- Font-size controls in the chat modal.

## Important: bring your own API key

This project does **not** provide any API endpoint or API key.

After installation, open the plugin settings and add an OpenAI-compatible chat completion endpoint:

- Endpoint example: `https://api.openai.com/v1/chat/completions`
- API key: your own key
- Model: the model name supported by your provider

Local gateways are supported as long as they expose an OpenAI-compatible `/v1/chat/completions` API.

## Manual installation

1. Download `main.js`, `manifest.json`, and `styles.css` from a release.
2. Create this folder in your vault:

   ```text
   .obsidian/plugins/pdf-chat/
   ```

3. Copy the three files into that folder.
4. Restart Obsidian or reload community plugins.
5. Enable **PDF Chat** in Settings → Community plugins.
6. Configure your model in **PDF Chat Settings**.
7. Bind or use the default command hotkey: `Ctrl/Cmd + Alt + Q`.

## Usage

1. Open a PDF in Obsidian.
2. Select a piece of text.
3. Run the command **PDF Chat: 针对选中内容提问 (PDF Chat)**.
4. Type your question and press `Enter`.

The plugin will use the selected text as immediate context. If the PDF text can be extracted, it can also use full-paper context or long-document retrieval depending on document size.

## Privacy

- API requests are sent to the endpoint you configure.
- PDF text and selected text may be sent to your configured model provider when you ask questions.
- Local caches such as `data.json` are generated by Obsidian and are intentionally ignored by Git.
- Do not commit your personal `data.json`.
