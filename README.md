# Obsidian PDF Chat

在 Obsidian 里划选 PDF（或其他可选中文字的视图）内容，按快捷键弹出聊天窗口，直接提问，答案在窗口里流式显示。

适合读论文：选中一段 → 提问 → 继续追问。**不提供任何公共 API**，你需要自己填自己的模型接口和 API Key。

## 1. 它能做什么

- 划选文字后弹出聊天框，`Enter` 提交，`Shift+Enter` 换行
- 流式输出（一边生成一边显示），可随时点「停止」
- 多轮对话记忆（关闭弹窗或重启 Obsidian 后，同一篇 PDF 也能继续上次对话）
- 用户提问和模型回答都可以用鼠标拖选并复制
- 可配置多个模型，弹窗里随时切换
- 内置「阅读模式」预设（速读地图、方法论解码等）
- 可选：自动生成全文摘要，作为回答背景
- PDF 不太长时：整篇读一遍再回答（更准）
- PDF 很长时：自动改用本地关键词检索（BM25），不依赖 embedding 服务
- 支持 Markdown / 公式显示，可调字体大小
- 聊天窗口可拖动、可拉伸

## 2. 安装（最重要）

### 方法 A：从 Release 下载（推荐）

1. 打开本仓库的 [Releases](https://github.com/anonymous369163/obsidian-pdf-chat/releases) 页面
2. 下载最新版的 `pdf-chat-x.x.x.zip`
3. 解压后应看到三个文件：
   - `main.js`
   - `manifest.json`
   - `styles.css`
4. 在你的 Obsidian 库里，找到（没有就新建）这个文件夹：

   ```text
   你的库/.obsidian/plugins/pdf-chat/
   ```

5. 把上面三个文件放进这个文件夹
6. 重启 Obsidian，或在「设置 → 社区插件」里点一下刷新
7. 打开「设置 → 社区插件」，启用 **PDF Chat**

### 方法 B：直接从源码拷贝

把仓库根目录的 `main.js`、`manifest.json`、`styles.css` 拷到同样的路径即可。

> 注意：不要把别人的、或自己的 `data.json` 一起发出去。`data.json` 里可能有 API Key、本地缓存和可见对话记录。

## 3. 第一次配置模型（必做）

插件装好后还不能直接用，必须先填自己的模型接口。

### 步骤

1. 打开 Obsidian **设置**
2. 左侧找到 **PDF Chat**
3. 在「模型列表」里编辑第一条（或点「添加模型」新建）
4. 填三个关键字段：

| 字段 | 填什么 | 例子 |
|------|--------|------|
| **显示名称** | 你自己看得懂就行 | `DeepSeek` / `本地网关` |
| **Endpoint** | 聊天补全接口地址，一般以 `/v1/chat/completions` 结尾 | 见下方示例 |
| **API Key** | 你自己的密钥 | `sk-...`（没有密钥的本地服务可留空） |
| **模型名(model)** | 服务端认的模型 ID | `deepseek-chat` / `gpt-4o-mini` / `qwen` |

5. 点「设为默认」（如果有多条）
6. 建议先把「摘要生成用的模型」也选成同一个或一个更快的模型
7. 关掉设置页，打开一篇 PDF 试一次

### Endpoint 怎么填（容易填错）

正确示例：

```text
https://api.deepseek.com/v1/chat/completions
https://api.openai.com/v1/chat/completions
http://127.0.0.1:11434/v1/chat/completions
http://你的网关地址:端口/v1/chat/completions
```

常见错误：

- 只填了域名，少了 `/v1/chat/completions`
- 填成了网页地址，而不是 API 地址
- 本地服务端口写错
- 公司/校园网关需要走代理，但 Obsidian 访问不到

> 本插件兼容 **OpenAI Chat Completions** 格式。只要对方提供类似接口，一般都能用。

## 4. 常用配置示例

下面只是示例，**请换成你自己的 Key 和地址**。仓库里不会、也不能提供可用的公共 Key。

### 示例 1：DeepSeek 官方

| 字段 | 建议填写 |
|------|----------|
| 显示名称 | DeepSeek |
| Endpoint | `https://api.deepseek.com/v1/chat/completions` |
| API Key | 你在 DeepSeek 平台申请的 Key |
| 模型名 | `deepseek-chat` |

### 示例 2：OpenAI 官方

| 字段 | 建议填写 |
|------|----------|
| 显示名称 | OpenAI |
| Endpoint | `https://api.openai.com/v1/chat/completions` |
| API Key | 你的 OpenAI Key |
| 模型名 | `gpt-4o-mini` 或其他你账号可用的模型 |

### 示例 3：本地 Ollama（OpenAI 兼容模式）

先确认 Ollama 已启动，并且开启了兼容接口。

| 字段 | 建议填写 |
|------|----------|
| 显示名称 | Ollama |
| Endpoint | `http://127.0.0.1:11434/v1/chat/completions` |
| API Key | 可留空，或随便填 `ollama` |
| 模型名 | 你本机已拉取的模型名，如 `qwen2.5:7b` |

### 示例 4：自建 / 校园 / 公司网关（new-api 等）

| 字段 | 建议填写 |
|------|----------|
| 显示名称 | 本地网关 |
| Endpoint | `http://IP:端口/v1/chat/completions` |
| API Key | 网关发给你的 Token |
| 模型名 | 网关里实际暴露的模型 ID（以网关 `/v1/models` 为准） |

可以添加多条模型配置，比如：

- 一条「DeepSeek」日常提问
- 一条「更快更便宜的模型」专门做全文摘要和检索词规划

## 5. 日常怎么用

1. 用 Obsidian 打开一篇 PDF
2. 用鼠标划选一段你想问的文字
3. 按快捷键（默认 `Ctrl + Alt + Q`，Mac 上是 `Cmd + Alt + Q`）
4. 弹出 PDF Chat 窗口
5. 在输入框里打问题，按 `Enter` 发送
6. 可以继续追问；用户提问和模型回答都能直接用鼠标拖选、复制
7. 关闭弹窗后再次从同一篇 PDF 打开，会恢复这篇 PDF 的上次对话；点「清空对话」会重新开始

同一篇 PDF 重新划选另一段文字时，会保留该 PDF 的对话历史，并把最新选区作为当前原文上下文。不同 PDF 的对话互不混用。非 PDF 视图只有在选中文字一致时才会恢复对应对话。

### 第一次打开某篇 PDF 时会发生什么

如果你开了相关开关（默认建议开启）：

1. 插件会尝试提取 PDF 文字
2. 可能自动生成一份「全文摘要」（用摘要模型，通常较便宜）
3. 建立本地检索索引（纯本地切块，不调用模型）
4. 你提问时：
   - PDF 不太长 → 把全文交给模型回答（更准）
   - PDF 很长 → 用关键词检索相关片段再回答

同一篇 PDF 第二次打开，一般会直接用缓存，不会每次都重新摘要。

## 6. 弹窗里有哪些按钮

| 区域 | 作用 |
|------|------|
| 标题栏拖动 | 按住标题可拖动窗口 |
| 右下角拉伸 | 可改变窗口大小 |
| **模型** 下拉框 | 切换当前回答用的模型 |
| **阅读模式** 下拉框 | 切换系统角色/提示词预设 |
| **A- / A+** | 缩小 / 放大聊天字体 |
| **附带全文摘要** | 是否把浓缩后的全文背景塞进系统提示 |
| **全文/检索相关片段** | 是否启用全文直读或长文检索 |
| **生成/刷新摘要** | 强制重新生成摘要 |
| **建立/刷新索引** | 强制重新切块建索引 |
| **清空对话** | 清空当前聊天记录，并删除当前 PDF/选区已保存的对话 |
| **发送 / 停止** | 发送问题；生成中变成停止 |

输入规则：

- `Enter`：发送
- `Shift + Enter`：换行

## 7. 设置页各项说明

打开：**设置 → PDF Chat**

### 7.1 模型列表

- 可添加多个模型（不同 endpoint / Key / 模型名）
- 其中一条可设为「默认」
- 弹窗里的「模型」下拉框会列出这里的全部条目

建议：

- 至少配置 1 个能正常聊天的主模型
- 可选再配 1 个更快的模型，专门给「摘要 / 检索角度规划」用

### 7.2 基础生成参数

| 设置项 | 通俗解释 | 建议 |
|--------|----------|------|
| 流式输出 | 一边生成一边显示 | 一般保持开启 |
| Temperature | 随机程度 | 读论文建议 `0.3 ~ 0.7` |
| Max Tokens | 单次回答最大长度 | 按需求调大，过大可能更慢更贵 |
| 系统提示词 | 默认角色说明 | 初学者可不改 |

### 7.3 全文摘要（浓缩上下文）

作用：先用一个模型把整篇论文压成短摘要，之后回答局部问题时带上这份背景。

| 设置项 | 通俗解释 |
|--------|----------|
| 自动附带全文摘要 | 打开弹窗时自动用缓存摘要；没有就自动生成 |
| 摘要生成用的模型 | 建议选便宜/快的模型 |
| 全文截断字符数上限 | 太长的 PDF 只拿前面一部分做摘要 |
| 摘要最大输出 token | 限制摘要本身别写太长 |
| 摘要生成提示词 | 控制摘要怎么写 |
| 清空已缓存摘要 | 删掉本地摘要缓存 |

### 7.4 全文直读 / 检索（RAG）

插件的策略很简单：

1. **短/中等长度 PDF**：直接读全文（更准）
2. **超长 PDF**：退回关键词检索（BM25）

| 设置项 | 通俗解释 |
|--------|----------|
| 全文直读的字数阈值 | 低于这个长度就直接读全文；默认约 `180000` 字符 |
| 自动建立检索索引 | 打开弹窗时自动切块建索引（本地，很快） |
| 提问前思考检索角度 | 中文问英文论文时很有用：先让快模型生成多组检索词 |
| Top K | 每次检索带回多少片段 |
| 单块最大字符数 / 重叠 | 切块参数，一般不用动 |
| 清空检索索引 | 删掉本地切块缓存 |

> 绝大多数单篇论文都会走「全文直读」。只有特别长的综述/教程才会走检索后备方案。

### 7.5 阅读模式预设

弹窗里的「阅读模式」来自这里。你可以增删改：

- 论文速读地图
- 方法论解码
- 局限与威胁
- ……

切换预设 = 换一套系统提示词，不会自动替你提问；你仍然要在输入框里写问题。

## 9. 常见问题排查

### Q1：按了快捷键没反应？

- 是否先划选了文字？没选中内容不会弹窗
- 插件是否已启用？
- 快捷键是否被其他插件占用？去「设置 → 快捷键」检查

### Q2：弹窗出现了，但请求失败 / Invalid token？

- API Key 是否填对、是否过期
- Endpoint 是否完整包含 `/v1/chat/completions`
- 模型名是否是你这个服务端真正支持的 ID
- 本地/内网地址是否能从你这台电脑直接访问（浏览器或 `curl` 测一下）

### Q3：回答感觉很慢？

- 第一次打开某篇 PDF，可能在生成摘要
- 模型本身慢，或 Max Tokens 设太大
- 网络到你的 API 服务较慢
- 长对话本身会带上历史；可点「清空对话」再问

### Q4：公式显示乱、挤在一起？

- 用弹窗里的 `A+` 稍微放大字体再看
- 确保答案是正常 Markdown / LaTeX 形式输出

### Q5：摘要老是截断？

- 提高「全文截断字符数上限」
- 或换上下文更长的模型做摘要
- 摘要本身应尽量短；太长反而干扰主问答

### Q6：中文问英文论文，检索不准？

- 先确认是否已开启「提问前先让快模型思考检索角度」
- 如果论文不算很长，优先依赖「全文直读」（默认阈值下多数论文都会走这条）

### Q7：我换了电脑，配置和对话历史还在吗？

- 配置和可见对话记录保存在库内：`.obsidian/plugins/pdf-chat/data.json`
- 同步库时如果同步了这个文件，配置和对话记录会一起带走
- **不要把这个文件公开上传到 GitHub**

## 10. 隐私与安全

- 你问问题、划选内容、以及可能的全文/摘要，都会发到**你自己配置的接口**
- 本插件作者不提供、也不中转任何 API
- 请妥善保管 API Key
- 开源仓库中的默认配置是空的，不会包含真实密钥
- 每篇 PDF 最近一次对话中的可见提问和回答会以明文保存在 `data.json`，直到你在对应弹窗中点击「清空对话」
- 对话记录不会重复保存 system prompt、自动附带的 PDF 全文或本轮 RAG 请求片段；但现有的摘要和本地检索缓存本身可能包含论文摘要或片段
- 请勿提交：
  - `data.json`
  - 含 Key 的截图
  - 内网地址 + Token 的组合

如果你曾经把真实 Key 写进过公开仓库或聊天记录，请立刻去对应平台**轮换/作废**该 Key。

## 11. 开发说明

本仓库是纯 JavaScript Obsidian 插件，无需构建：

```text
main.js
manifest.json
styles.css
README.md
LICENSE
versions.json
```

本地测试：把这三个插件文件放到：

```text
<你的库>/.obsidian/plugins/pdf-chat/
```

无需安装依赖即可运行仓库内的回归与语法检查：

```bash
node --test tests/conversation-persistence.test.js
node --check main.js
```

然后重载社区插件即可。
