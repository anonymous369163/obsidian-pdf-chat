# PDF Chat 0.8.2 Reliability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Preserve reading state across vault changes and keep long API conversations bounded, explainable, and extraction-aware.

**Architecture:** Add pure reconciliation, context-composition, and extraction-quality modules, then inject small service interfaces into the existing plugin/modal. Keep 0.8.1 persistence compatible in this release so the later 0.8.3 repository migration is isolated.

**Tech Stack:** TypeScript 5.8, Obsidian API 1.8, esbuild, Node built-in test runner.

## Global Constraints

- No new runtime dependency or UI framework.
- Public release remains `main.js`, `manifest.json`, and `styles.css`.
- Never persist absolute PDF paths, selected passage text, hidden RAG/full-text payloads, or credentials outside their existing approved settings location.
- Every behavior change follows RED → GREEN → refactor.
- Existing `data.json` remains readable and deploy-local must preserve it.
