# PDF Chat 0.8.3 Layered Storage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move long-lived sessions/tasks and large paper caches out of monolithic `data.json` into recoverable per-entity JSON repositories without losing 0.8.1/0.8.2 user data.

**Architecture:** Introduce an adapter-backed atomic JSON store, then build session and paper repositories on it. A checkpointed migration hydrates repositories before stripping legacy large fields from settings; existing services consume repository interfaces rather than file paths.

**Tech Stack:** TypeScript, Obsidian `DataAdapter`, Node built-in test runner, esbuild; no database or new runtime package.

## Global Constraints

- Migration must never copy API keys, endpoints, model profiles, or prompts into `reader-data`.
- A failed or interrupted migration leaves legacy settings usable.
- Entity writes validate before replacing the last good file.
- Sessions/evidence are never quota-evicted; only regenerable paper assets are eligible.
- Public release remains exactly three files and deploy-local preserves all local JSON data.

```ts
export interface JsonAdapter {
  exists(path: string): Promise<boolean>;
  read(path: string): Promise<string>;
  write(path: string, data: string): Promise<void>;
  rename(from: string, to: string): Promise<void>;
  remove(path: string): Promise<void>;
  mkdir(path: string): Promise<void>;
}

export class AtomicJsonStore<T> {
  constructor(
    private adapter: JsonAdapter,
    private path: string,
    private validate: (value: unknown) => T
  ) {}
  readWithBackup(): Promise<T | null>;
  write(value: T): Promise<void>;
}
```

Serialize writes per instance. Write `.tmp`, parse and validate it, rotate the validated primary to `.bak`, then rename `.tmp` to primary. Redact paths from public errors when they are absolute.
