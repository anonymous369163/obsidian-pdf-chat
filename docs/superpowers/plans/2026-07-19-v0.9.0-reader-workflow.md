# PDF Chat 0.9.0 Reader Workflow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn durable conversations into navigable evidence, Obsidian research notes, manageable session collections, and recoverable cross-device Codex work.

**Architecture:** Version messages and sessions for stable identity, add isolated evidence/note/library services, then expose their actions through compact message footers and a dedicated modal. Cross-device recovery creates an explicit child session/new thread; future research/PPT abilities register through credential-blind capability contracts.

**Tech Stack:** TypeScript, Obsidian DOM/Vault APIs, existing Markdown renderer, Node built-in test runner, esbuild.

## Global Constraints

- No React, database, OCR engine, online provider, or PPT runtime in this release.
- Evidence never becomes `located` without a known PDF and valid page.
- Notes/exports contain visible content and relative vault links only.
- Session deletion never deletes PDFs or notes and is blocked while a managed Codex turn runs.
- Cross-device recovery never pretends to resume a missing native thread.
- Capability adapters cannot receive plugin settings or credentials.
